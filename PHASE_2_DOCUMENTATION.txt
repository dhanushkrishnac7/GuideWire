1. OVERVIEW & WHAT HAS CHANGED
---------------------------------------------------
In Phase 1, Humsafar established the foundational structure for an income protection platform for gig workers. In Phase 2, the application underwent a complete architectural and visual redesign focusing on **Zero-Touch Automation**, **Adversarial Fraud Defense**, and **Dynamic ML Risk Pricing**.

Key differences from the initial phase/README:
- UI Overhaul: Migrated from a bare-bones dark theme to a bright, conversion-optimized Light Theme mimicking an enterprise fin-tech application. The screens have been redesigned 1:1 with high-fidelity mockups.
- Dynamic Live Context: Previously, the Home Screen displayed hardcoded mock text. It now actively fetches live, real-time data streams from a global Context Provider mapping simulated events to real updates.
- Adversarial Defense: Replaced standard verification steps with an ML Isolation Forest Anomaly Detection pipeline that actively checks the worker's device telemetry for GPS Spoofing and mock locations.
- Interactive Simulator Engine: Replaced static API testing. Allowing users/judges to dynamically push thresholds (Rain: mm/hr, Heat: ºC, AQI: Index, Floods: Alert Status) and immediately trace the effect on the payout platform logic.

2. CORE FEATURES & MODULES
---------------------------------------------------
A. Onboarding & Policy Selection (Insurance Management)
- Users quickly designate their fleet (Food Delivery, Logisics) and bind their Delivery ID.
- Dynamic AI Risk Intelligence provides hyper-local exposure levels (e.g., Bengaluru South marks a 78% risk).
- Pricing Model: Evaluates local environmental factors to charge appropriately (Humsafar Premium / Standard Safety) tied exclusively to a weekly recurring billing cycle.

B. Parametric Verification Engine (Zero-Touch Claims)
- Event listeners evaluate mock API endpoints (Weather/Traffic/Flood) against rigid AI thresholds (e.g. >50mm/hr of rain).
- No manual insurance claim filing is allowed. The framework handles the logic, verifying the active hours of the delivery driver within the affected geofence, avoiding fraud.

C. Intelligent Fraud Defense 
- Sits natively in the Zero-Touch Pipeline. Evaluates the "Location Activity Confirmed" checkpoint.
- If a Mock Location app signature or unusually fast anomalous motion is detected through Isolation Forest validation, the pipeline halts mid-verification, blocking the UPI payout globally.

4. SIMULATION PIPELINE GUIDE FOR TEAMMATES
---------------------------------------------------
The dedicated Simulation Screen acts as our localized API Mock connector for recording demo videos or internal testing. 
- You manually adjust the sliders (Rain, Heat, AQI) to breach the parametric thresholds (e.g. Slide to >45ºC).
- Press "Execute Zero-Touch Flow" to fire a Live Trace. The UI will step through parametric verification.
- GPS SPOOFING TEST: Near the bottom, checking the "Adversarial Defense: Simulate GPS Spoofing" checkbox will intentionally fail Step 2 of the pipeline above, explicitly rejecting the transaction to prove our Anti-Fraud tech to the judges!
